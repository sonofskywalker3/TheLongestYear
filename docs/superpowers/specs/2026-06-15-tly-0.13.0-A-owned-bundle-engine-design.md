## Where this sits

0.13.0 (the difficulty engine) was decomposed into 4 sub-projects, to be specced/built in order:
- **A — Owned-bundle engine (THIS DOC).** Foundation: TLY owns + per-loop re-randomizes its own bundle set, replacing the read-from-game path. Single default difficulty (vanilla-equivalent "Normal" baseline).
- **B — Difficulty tiers (Easy/Normal/Hard).** Parameterize A. Tier chosen at run creation.
- **C — Cart economy by tier.** Easy untouched; Normal = the JP-permanent cart slots already shipped in 0.12.0; Hard = gold-per-run slots + JP-lock.
- **D — Brutal Hard gate content + escape valves.** Galaxy Sword, 150 Salmonberries, artifact-any-museum, iridium roe, etc.; winnable via cart RNG / JP cultivation perks / stash carry.

(Seasonal "retaliation" escalations = 0.14.0, separate.)

Full roadmap context: user memory `tly-012-013-014-roadmap.md`. 0.12.0 (clarity + cart economy + Cart Whisperer tiers) is code-complete at manifest **v0.11.10**, local-only, unreleased.

## Current architecture (what A replaces)

- Today TLY READS the game's live bundles: `BundleCatalogBuilder.Build()` / `.BuildRequirements()` read `Game1.netWorldState.Value.BundleData`, parse via `BundleParsing`, classify rooms via `RoomThemeMap` (Pantry→Farming, CraftsRoom→Foraging, FishTank→Fishing, BoilerRoom→Mining, Bulletin/BulletinBoard→Mixed; Vault/Joja/unknown skipped). This read-and-classify path is the source of the remix-fragility bugs (#2 remix bundles missing from checkpoints, #4 impossible mining theme, blank weekly themes).
- Downstream consumers of the classified set: `BundleRequirement` (carries `Ingredients`, `NumberOfSlots`, `IngredientStacks`, `IngredientQualities`), `BundleGate.IsSatisfied`, the weekly-theme service, the season-goal checklist (`SeasonGoalsMenu`), and `CcDonationReconciler` (derives donated ids from the game's authoritative per-slot completion).
- Reset path: `RunController.FinalizeReset` → `WorldResetService.PerformReset`; `CommunityCenterUnlock.Apply()` already calls `Game1.netWorldState.Value.SetBundleData(...)` + `cc.MakeMapModifications(force:true)` and pre-reveals all 6 Junimo Notes.
- TLY is dormant on non-TLY saves (`RunActivation.IsActive`); per-save opt-in (`MetaState.IsLongestYearRun`); `MetaState.CompletedResets` = loop counter.

## DECISIONS so far (confirmed with user)

**Generation model = "expanded-pool remix":**
1. Every room **keeps its theme** (reuse `RoomThemeMap`).
2. Each room has a **bundle pool** = the existing vanilla bundles **+ new TLY-authored bundles**. Each loop, **remix which bundles appear** (vanilla-style remix, but drawing from the bigger pool incl. the new ones).
3. Each bundle keeps its **slot count**, but slots are filled from an **expanded, theme/season-appropriate item pool**, wider than vanilla, **re-randomized every loop**. Examples the user gave: *Spring Crops* can pull ANY spring-growable crop (incl. harder/year-2-ish); *Quality Crops* can ask for various vegetables, not the fixed vanilla four.
4. The "keep a little of everything across runs" meta comes from this per-loop content re-roll (you can't predict next loop's asks).

**Item pool sourcing = HYBRID:**
- **Existing bundles' expanded pools → derived from game data** at runtime (e.g. `Data/Crops` by season, fish data by location/season, forage tables). Auto-includes year-2 + modded items; needs an exclude-list for junk/quest/un-obtainable items.
- **New bundles → hand-authored item PATTERNS** (curated pools), recognized/created by us. Confirmed new bundles:
  - **Artifact bundle → Bulletin Board (Mixed theme).** Pattern: can roll museum/artifact donation items (ties to the earlier "stockpile artifact troves across runs, gamble on contents" idea).
  - **Gemstone bundle → Furnace/Vault (Mining theme or Vault room).**
  - (More new bundles may be added.)

**Difficulty hook (deferred to B, NOT in A):** Hard weights pools toward rarer/harder items and/or bumps required quantities. A ships a single vanilla-equivalent baseline.

## OPEN QUESTIONS (resolve next, one at a time, in the brainstorm)

1. **Write mechanism + re-randomization timing.** How does TLY get its generated bundles into the real CC so the vanilla `CommunityCenter` menu + completion just work?
   - (a) Edit `Data/Bundles` via `Helper.Events.Content.AssetRequested` (then invalidate/reload per loop), or
   - (b) Directly write `Game1.netWorldState.Value.BundleData` + `SetBundleData(...)` at run-create and on each reset.
   - (b) looks more natural for *per-loop* re-randomization (it's already how `CommunityCenterUnlock` seeds bundle data), but confirm interaction with `Data/Bundles` and the game's own "Remixed bundles" new-game toggle (TLY now overrides it).
2. **CC completion-state reset on re-randomization.** Regenerating bundles each loop must also reset `CommunityCenter.areasComplete` + per-bundle completion arrays so a new loop's bundles start empty. Coordinate with `FinalizeReset`/`WorldResetService`/`CommunityCenterUnlock`.
3. **Determinism / seeding.** Per-loop seed (e.g. save seed + `CompletedResets`) so a save reload does NOT reshuffle the current loop's bundles. (Vanilla uses `Utility.CreateDaySaveRandom`-style seeding — mirror a stable per-loop seed.)
4. **Downstream wiring.** Point `BundleCatalogBuilder.BuildRequirements` (and the gate, weekly themes, season checkpoints, `IngredientStacks`/`IngredientQualities`) at TLY's generated set instead of read-from-game. Confirm this auto-resolves #2/#4/#6 and the quality/quantity surfacing. Keep `CcDonationReconciler` working (it reads the game's per-slot completion, which now reflects TLY's bundles).
5. **New-bundle authored patterns — detail.** Define artifact bundle + gemstone bundle (slot counts, item patterns, room/theme, reward), and decide how many new bundles per room and how the remix picks them.
6. **Game-data pool derivation specifics.** How exactly to query "all spring crops" / "all river fish in season" / forage; what "year-2 stuff" includes; the exclude-list contents; how season-gating maps to the bundle's room/season.
7. **Existing-run migration.** Do owned bundles apply only to new runs, or regenerate for in-progress runs on their next loop?
8. **Slot quantity/quality in generated bundles.** Tie to the existing `IngredientStacks`/`IngredientQualities` the 0.12.0 `SeasonGoalsMenu` already renders.
